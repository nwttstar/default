SimpleWebTask

このプロジェクトに必要なこと

  • 動作が軽く、機能が必要最低限であること
  • タスクを走らせる十分な意義があること
  • 設定が容易であり、カスタムがしやすいこと

動作環境

バージョン

SimpleWebTask v1.0 (2020/09/10)

プロジェクトについて

クローン後は、ルートディレクトリにてベーシックなWeb制作をしていただけます。
ディレクトリ構造については次のセクションに記載しています。

プロジェクトにはすでにタスクランナー(gulp)、scssのコンパイル(gulp-sass)・JSのバンドル(webpack)・JSのES5コンパイル(babel)という必要最低限の機能が備わっています。

タスクランナー(gulp)を走らせるには Node.js/ にて、ターミナルからコマンドを打ち込みます。
コマンドはpackage.jsonの記述を参照。

ディレクトリ説明

ディレクトリ構造

root/ 　├ index.html 　├ _Assets/ 　│　├ index.css 　│　└ index.js 　└ _Develops/ 　　　├ Node.js/ 　　　│　├ node_modules/ 　　　│　├ gulpfile.js ★ 設定ファイル 　　　│　├ package.json 　　　│　└ webpack.config.js ★ 設定ファイル 　　　└ Resources/ 　　　　　├ js/ 　　　　　├ scss/ 　　　　　└ views/

構成について

このプロジェクトは _Develops/ > Node.js/ 内で動く、Node.js環境です。
npm run スクリプトはpackage.jsonが存在する Node.js/ フォルダ内で走らせます。

作業者は主に root/下にフォルダを作成したり、 _Assets/ フォルダ内や、★ 設定ファイルを触ります。

各ディレクトリの説明

役割がわかりやすいように分類わけしてあります。

root/

このプロジェクトではルーティングを行わないので、こちらがウェブサイトのルートディレクトリになります。
index.html が初期ページです。

_Assets/

HTML内で読み込むファイルを収納するディレクトリです。画像や動画、pdfなどを分類わけします。
また、コンパイルされたjsやcssもこちらに吐き出されます。
既定ではそれぞれ index.css index.js と命名されます。

HTML内で指定されるパス(src属性)は、 ./_Assets/.. から始まるものが望ましいです。

<img src="./_Assets/images/PC/titlelogo.png" alt="title"> など。

_Develops/

主にこちらで開発を進めます。Developは開発の意。

Node.js/

npm run スクリプトを走らせるには、このディレクトリに移動します。

中身はディレクトリ名の通りです。Node.jsに関するモジュールや、モジュールの設定などはこちらにあります。
gulpやbabelはNode.jsで動いています。

gulpfile.js webpack.config.js といった設定ファイル内で定義されたconfig のプロパティを変更することで、
jsやscssのコンパイルを行うgulpやwebpack(bandle,babel)の挙動を変更できます。

この中をいじるときは、主にディレクトリ構造を変更するときと、nodeモジュールを追加するときだけ。

Resources/

コンパイル前のファイルを作るところ。scssやjsはこちらに作成します。
gulpはこのディレクトリを監視し、コンパイル用のタスクを走らせます。

Node.jsはスクリプトを起動することでこのディレクトリのファイルを監視・コンパイルし、コンパイル後のファイルを _Assets/ へと吐き出します。

なお、jsはwebpackを知っている前提です。エントリーポイントは Resources/js/index.js に設定してあります。
views/ にはテンプレートをしまいます。使用しない場合は必要ありません。

作業する

_Assets/ にWeb制作で使用する画像や動画を収納します。
また、ルート下に自由にディレクトリを作成できます。

作業時にはタスクランナー(gulp)を起動させます。
_Develops/ > Node.js/ のディレクトリにて、ターミナルを開きます。

まずは npm i というコマンドを実行し、必要なモジュールをインストールします。
その後、以下のスクリプトを使えるようになります。（同ディレクトリ）

gulp
Resources/ > SCSS/ を監視し、scssコンパイルを行います。
gulpをグローバルインストールしている必要がある。
npm run build
Resources/ > SCSS/ と Resources/ > JS/にて、scssコンパイル・JSのバンドル・JSのES5コンパイルを行います。
npm run scss
Resources/ > SCSS/ にて、scssコンパイルを行います。
npm run js
Resources/ > JS/ にて、JSのバンドル・JSのES5コンパイルを行います。
npm run dev
Resources/ > SCSS/ と Resources/ > JS/ を監視し、scssコンパイル・JSのバンドル・JSのES5コンパイルを行います。

デプロイについて

Web制作では開発を進めるごとにファイル数が嵩んでいきます。FTPなどでデプロイをする際、増えたデータの分だけ転送データを選ぶプロセスが重くなります。
このプロジェクトではディレクトリ構造を単純化することで、その問題点を解決しています。

HTMLで呼び出すデータを _Assets/ 、ステージング環境に使用しない開発データを_Develops/ と分けることで、デプロイを単純化できます。

具体的にはステージング環境に _Develops/ を転送しないことで実現できます。 (あるいは _Develops/ 以外すべてを転送する）
